💡 PRO TIPS
═══════════════════════════════════════════════════════════════════════════════

1. Keep Terminal Open
   While developing, keep npm start running in one terminal
   and use another for git commands

2. Browser DevTools
   Use F12 → Console to check for errors
   Use F12 → Network to verify API calls

3. Test Before Deploy
   Always test locally before deploying to cloud
   Saves time debugging production issues

4. Environment Variables
   Keep .env secure
   Never commit it
   Use .env.example as template

5. Deployment Order
   Backend first (takes longer)
   Frontend second (faster)
   This way frontend can be updated easily if needed
